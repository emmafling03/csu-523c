Question 2: Visualization

Natural Color (RGB) — What the Human Eye Would See

This picture shows the image as it would look to the human eye, making it easy to identify major land features like rivers, roads, and urban areas.

plotRGB(landsat_crop, r = 4, g = 3, b = 2, stretch = "lin", 
        main = "Natural Color: Red-Green-Blue")

Color Infrared (Vegetation)

This composite highlights vegetation in bright red because “healthy” plants strongly reflect near-infrared light. Water and urban areas appear darker, making it useful for distinguishing vegetation health and general land cover.

plotRGB(landsat_crop, r = 5, g = 4, b = 3, stretch = "lin",
        main = "Color Infrared: NIR-Red-Green\n(Bright red = healthy vegetation)")

Water-Focused False Color

This combination makes water detection easier because water absorbs strongly in both NIR and SWIR bands, so flooded or open water areas appear very dark or distinct blue and black. It can be useful for mapping flood extent and separating water from moist soil or vegetation.

plotRGB(landsat_crop, r = 5, g = 6, b = 4, stretch = "lin",
        main = "Water Focus: NIR-SWIR1-Red\n(Dark blue = open water/flooding)")

NDVI

This highlights vegetation health by comparing near-infrared and red reflectance, where healthy vegetation appears in higher values. Areas with little or no vegetation, such as water or bare soil appear darker which can make it useful for identifying vegetated vs. non-vegetated areas

ndvi  <- (landsat_crop$nir08 - landsat_crop$red) / (landsat_crop$nir08 + landsat_crop$red)

plot(ndvi, col = colorRampPalette(c("blue", "white", "red"))(256))

Normalized Difference Water Index (NDWI)

This index makes surface water easy to see by comparing green and near-infrared reflectance, where water typically appears as high (bright) values. It provides a clear separation between flooded areas and dry land,and can be used to delineate flood extent.

ndwi <- (landsat_crop$green - landsat_crop$nir08) /
        (landsat_crop$green + landsat_crop$nir08)

plot(ndwi,
     col = colorRampPalette(c("brown", "white", "blue"))(256),
     main = "NDWI: Water/Flood Detection\n(Blue = water, brown = dry land)")

Step 2: Thresholding

NDVI <0 = Water

ndvi_flood <- app(ndvi, function(x) ifelse(x < 0, 1, 0))

NDWI > 0 = water

ndwi_flood <- app(ndwi, function(x) ifelse(x > 0, 1, 0))

MNDWI > 0 = water

mndwi_flood <- app(mndwi, function(x) ifelse(x > 0, 1, 0))

WRI > 1 = water

wri_flood <- app(wri, function(x) ifelse(x > 1, 1, 0))

SWI < 5 = water

swi_flood <- app(swi, function(x) ifelse(x < 5, 1, 0))
Stack and Name
flood_stack <- c(ndvi_flood, ndwi_flood, mndwi_flood, wri_flood, swi_flood)

names(flood_stack) <- c("NDVI", "NDWI", "MNDWI", "WRI", "SWI")
Clean NA
flood_stack <- app(flood_stack, function(x) ifelse(is.na(x), 0, x))
Plot
plot(flood_stack, col = c("white", "blue"))

Step 3

All five maps appear similar because the thresholding process converts each index into a binary classification, where pixels are simply labeled as water or non-water. As a result, the main river channel and nearby flooded areas are consistently identified across all maps. The similarities are strongest in the river, which is clearly delineated in every method. However, small differences exist in the extent and distribution of scattered flooded pixels away from the river, with some maps showing more isolated patches than others. These differences reflect how each index responds slightly differently to surrounding land cover, even though they capture the same overall flood pattern.
